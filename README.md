 Өгөгдлийн сангийн төсөл – Спорт заалын удирдлагын систем
 Системийн хүснэгтүүд
gym_db өгөгдлийн сан нь дараах үндсэн хүснэгтүүдээс бүрдэнэ:
- Хүснэгт 1 – members (Гишүүд)
- Хүснэгт 2 – trainers (Дасгалжуулагчид)
- Хүснэгт 3 – subscriptions (Захиалга/бүртгэл)
- Хүснэгт 4 – attendance (Ирц)
Эдгээр хүснэгтүүд нь Foreign Key ашиглан харилцан холбоотой.

 Ашиглах заавар
Өгөгдлийн сантай ажиллах дараалал:
- MySQL серверийг суулгаж, gym_db өгөгдлийн санг үүсгэнэ
 файлыг ажиллуулж хүснэгтүүдийг үүсгэнэ
- Жишээ өгөгдлийг оруулж, хүснэгтүүдийн бүтцийг шалгана

4-р хэсэг: онол (Security)
1. CHECK constraint гэж юу вэ? Ямар нөхцөлд ашигладаг вэ?
- CHECK constraint нь өгөгдлийн сангийн хүснэгтэд оруулах утгууд тодорхой нөхцөлийг хангаж байгаа эсэхийг шалгадаг хязгаарлалт юм.
- Жишээ нь: нас нь 0-ээс бага байж болохгүй, эсвэл цалин нь 500,000-аас их байх ёстой гэх мэт.
- Жишээ:
CREATE TABLE Employees (
    EmpID INT PRIMARY KEY,
    Name VARCHAR(50),
    Age INT CHECK (Age >= 18),
    Salary DECIMAL(10,2) CHECK (Salary > 0)
);


Энд Age багана нь 18-аас бага утга авахгүй, Salary багана нь 0-ээс их байх ёстой гэсэн нөхцөлтэй.

2.DISTINCT түлхүүр үг ямар үүрэгтэй вэ?
- DISTINCT нь SQL асуулгад давхардсан мөрүүдийг арилгаж, зөвхөн давтагдахгүй утгуудыг буцаадаг.
- Энэ нь өгөгдлийн сангийн дотор олон давхардсан утга байхад зөвхөн өвөрмөц утгуудыг харах шаардлагатай үед хэрэгтэй.
- Жишээ:
SELECT DISTINCT Department
FROM Employees;


Энэ нь ажилчдын хүснэгтээс давтагдахгүй хэлтсүүдийн нэрсийг гаргаж авна.

 Authentication буруу тохируулах эрсдэл
3.Өгөгдлийн сангийн нэвтрэх эрхийг зөв тохируулахгүй бол дараах эрсдэлүүд үүснэ:
- Зөвшөөрөлгүй хандалт: Гадны хүн өгөгдлийн сан руу нэвтэрч, мэдээллийг өөрчилж эсвэл устгаж болзошгүй.
- Мэдээлэл алдагдах: Хувийн болон нууц мэдээлэл (жишээ нь хэрэглэгчийн нэр, нууц үг, санхүүгийн мэдээлэл) алдагдах эрсдэлтэй.
- Өгөгдлийн integrity алдагдах: Зөвшөөрөлгүй хэрэглэгчид өгөгдлийг буруу өөрчилснөөр системийн найдвартай байдал алдагдана.
- Аюулгүй байдлын зөрчил: Хакерууд өгөгдлийн санг ашиглан бусад систем рүү халдлага хийх боломжтой.

Товчхондоо:
- CHECK constraint нь өгөгдлийн зөвшөөрөгдөх утгыг хязгаарлана.
- DISTINCT нь давхардсан утгыг арилгана.
- Authentication буруу тохирвол өгөгдлийн сангийн аюулгүй байдалд ноцтой эрсдэл үүснэ.
